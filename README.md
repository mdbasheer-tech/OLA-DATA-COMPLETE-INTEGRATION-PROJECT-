Objective
- Analyze 100,000 ride booking records
- Clean and transform raw data
- Solve structured business questions using SQL
- Build an interactive dashboard in Power BI

Tools Used
- Excel
- SQL
- Power BI


Data Preparation
In Excel
- Removed duplicates
- Handled missing values
- Standardized date and time formats
In Power BI
- Corrected data types
- Created calculated columns
- Built measures for KPIs

Main Dataset Columns
- Date & Time
- Booking_ID
- Booking_Status
- Customer_ID
- Vehicle_Type
- Pickup & Drop Location
- Ride_Distance
- Booking_Value
- Payment_Method
- Driver_Ratings & Customer_Rating
- Cancellation & Incomplete Ride Reasons

Business Questions Solved (SQL)
- Retrieve all successful bookings
- Find the average ride distance
- Get the total number of canceled rides by customers
- Identify the top 5 customers with the highest number of rides
- Find the number of rides canceled by customers due to personal and car-related issues
- Find the maximum and minimum driver ratings for the Prime Sedan vehicle type
- Retrieve all payments made using UPI
- Calculate the average customer rating per vehicle type
- Calculate the total booking value of successfully completed rides
- List all incomplete rides along with reasons for incompletion
Note: SQL was used extensively for filtering, grouping, aggregations, and extracting business insights.


Interactive Business Dashboard (Power BI)
- Ride Volume Over Time
- Booking Status Breakdown
- Revenue by Payment Method
- Top 5 Customers
- Cancellation Analysis
- Ratings Analysis

Key Insights
- Most bookings are completed successfully
- Prime Sedan shows strong driver ratings
- UPI is a commonly used payment method
- Top 5 customers contribute significant revenue
- Cancellation reasons highlight operational gaps

Conclusion
- Successfully completed an end-to-end analytics workflow
- SQL enabled solving structured business questions
- Power BI provided interactive visualization capabilities
- Extracted actionable insights to support decision-making
